# Simple Task List API

A GraphQL API for managing tasks, built with Node.js, TypeScript, GraphQL Yoga, Pothos, Prisma, and Zod.

## Assignment context

This repository is my take-home coding assignment submission for the **Junior Backend Engineer** role at **Lush Handmade Cosmetics**.


## Tech stack

- Node.js
- TypeScript 
- GraphQL Yoga
- Pothos GraphQL
- Prisma 
- Zod

## Features

- `Task` data model with:
  - `id`
  - `title`
  - `completed`
  - `createdAt`
  - `updatedAt`
- Queries:
  - `tasks`
  - `task`
- Mutations:
  - `addTask`
  - `toggleTask`
  - `deleteTask`
- Bonus mutation:
  - `updateTask`
- Input validation using Zod
- Basic error handling with `GraphQLError`

## Requirement coverage

- **Task model in Prisma:** implemented in `prisma/schema.prisma` with `id`, `title`, `completed`, `createdAt`, and `updatedAt`
- **Pothos Task object type:** implemented in `src/schema/task.type.ts`
- **Required queries:** `tasks`, `task` in `src/schema/queries.ts`
- **Required mutations:** `addTask`, `toggleTask`, `deleteTask` in `src/schema/mutations.ts`
- **Zod input validation:** all operations with inputs validate args via schemas in `src/schema/validators.ts`
- **Schema generated by Pothos:** `builder.toSchema()` in `src/index.ts`
- **Basic error handling:** invalid inputs and missing mutation targets return GraphQL errors
- **Bonus (optional):** `updateTask` mutation

## Setup

1. Install dependencies:

```bash
npm install
```

2. Generate Prisma client (and Pothos Prisma types):

```bash
npx prisma generate
```

3. Push schema to local SQLite database:

```bash
npx prisma db push
```

## Run the API

Start development server:

```bash
npm run dev
```

Build for production:

```bash
npm run build
```

Start built server:

```bash
npm run start
```

GraphQL endpoint:

`http://localhost:3000/graphql`

## Example operations

### Query all tasks

```graphql
query {
  tasks {
    id
    title
    completed
  }
}
```

### Query a single task

```graphql
query {
  task(id: "TASK_ID") {
    id
    title
    completed
  }
}
```

### Add a task

```graphql
mutation {
  addTask(title: "Go shopping") {
    id
    title
  }
}
```

### Toggle a task

```graphql
mutation {
  toggleTask(id: "TASK_ID") {
    id
    completed
  }
}
```

### Delete a task

```graphql
mutation {
  deleteTask(id: "TASK_ID") {
    id
  }
}
```

### Bonus: update a task title

```graphql
mutation {
  updateTask(id: "TASK_ID", title: "Updated title") {
    id
    title
    completed
  }
}
```

## Notes

- Task IDs are generated as CUID strings.
- `createdAt` is set automatically on create.
- `updatedAt` is updated automatically on each update.
- `task(id)` returns `null` when a task does not exist, and returns a GraphQL error for invalid id format.
